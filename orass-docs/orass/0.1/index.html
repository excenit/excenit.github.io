Introduction to Excenit RSS

Overview

Welcome to Excenit RSS. This documentation covers how to use all aspects of the system to generate and submit returns to BoG. Excenit RSS is an automated, end-to-end solution for managing returns. It interfaces directly with BoG’s ORASS API to retrieve pending returns, manage report generation including validation, approval of returns, submission to BoG and status updates of submitted returns.

The documentation has two main parts - functional, which covers how to generate and submit returns to BoG, and technical, which covers how to manage the entire system.

Functional Documentation

The functional documentation explains how to generate and submit returns. There five main tasks:

  • Data loading

  • Dava validation

  • Report generation

  • Report review

  • Data analytics

Each of these tasks is explained in detail in the documentation.

Technical Documentation

The technical documentation explains how to manage the entire system. It covers security, data management, audit logs, notification system, and many others.
